# Krunker Constraint Editor

> **Note:** This README was generated by AI.

A browser userscript that adds an in-editor TAS/pathfinding constraint tool to the [Krunker.io](https://krunker.io) map editor. It lets you visually place and manage movement constraints (checkpoints, jump arcs, corridors, and more) directly in the 3D scene, then export them as JSON for use in TAS or pathfinding workflows.

## Features

- 15 constraint types: `HardCheckpoint`, `JumpArc`, `Corridor`, `LineSegment`, and more
- Click-to-place constraints using 3D raycasting
- Undo/redo, duplicate, import/export as JSON
- Auto-saves per map via `localStorage`
- Keyboard shortcuts: `C` toggle UI · `A` add · `Delete` remove · `Ctrl+Z/Y` undo/redo · `Ctrl+D` duplicate

## Requirements

- [Bun](https://bun.sh) (package manager & runtime)
- A userscript manager such as [Tampermonkey](https://www.tampermonkey.net)

## Setup

```bash
# Install dependencies
bun install

# Build the userscript
bun run build
# Output: build/krunker-constraint-editor.user.js
```

## Development


## Installation

1. Build the script with `bun run build` (or use the served URL in watch/serve mode).
2. Open Tampermonkey → **Create a new script**.
3. Paste the contents of `build/krunker-constraint-editor.user.js` and save.
4. Navigate to `https://krunker.io/editor.html` – a **Constraints** button will appear in the bottom-right corner.

## Usage

1. Open the Krunker map editor and click **Constraints** (bottom-right) or press `C`.
2. Press `A` to enter placement mode, then click in the scene to place a constraint.
3. Select a constraint in the list to edit its properties.
4. Use **Export** to save constraints as JSON, or **Import** to load a previously saved file.
